Optimal Ordering Example

The head elements above follow the optimal order defined by capo.js.

Benefits of Proper Ordering:

  • Critical metadata parsed first
  • Async resources loaded early
  • Render-blocking resources properly positioned
  • Prefetch/prerender happens last (lowest priority)
  • Improved page load performance

Weight Hierarchy:

Weight Element Type Why
10 META Critical metadata (charset, viewport, CSP)
9 TITLE Important for SEO and browser tabs
8 PRECONNECT Start connections early
7 ASYNC_SCRIPT Non-blocking scripts
6 IMPORT_STYLES CSS @imports block rendering
5 SYNC_SCRIPT Blocking scripts
4 SYNC_STYLES Blocking stylesheets
3 PRELOAD Hint for upcoming resources
2 DEFER_SCRIPT Scripts that wait for DOM
1 PREFETCH Low priority future resources
0 OTHER Everything else
